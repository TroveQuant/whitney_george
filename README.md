# 惠特尼乔治策略

这是一个基于 A 股市场的“惠特尼乔治”风格选股与跟踪项目，核心目标是将原始投资思路做成一个可复现、可回测、可每日自动更新、可生成报告并邮件推送的策略系统。

## 1. 策略简介

- **策略名称**：惠特尼乔治策略
- **市场范围**：A 股全市场（剔除北交所、ST、停牌股、上市未满一年的新股）
- **资产类别（Asset Class）**：股票（Equity）
- **策略类型（Strategy Type）**：
  - 价值型（Value）
  - 质量型（Quality）
  - 小市值/中小盘偏好（Small Cap Bias）
  - 多因子打分选股（Multi-factor Stock Selection）
  - 多头股票组合（Long-only Equity Portfolio）

本项目包含两套思路：

1. **华创复现版**
   - 按参考研报/思路做因子复现；
   - 主要从市值、杠杆、自由现金流、ROA、ROIC、PE、PB、PS 等维度筛选。

2. **增强版**
   - 在复现基础上加入更明确的阈值约束；
   - 更偏向实盘跟踪与工程化输出；
   - 当前主程序默认运行的是 **增强版**。

## 2. 投资逻辑

策略整体偏向“便宜且质量不差的小市值公司”，重点关注以下方向：

- 总市值较小；
- 资产负债结构相对稳健；
- 盈利能力和资本回报较好；
- 估值不过高；
- 股价不过高；
- 股息率具备一定支撑。

从代码实现看，当前增强版核心筛选维度包括：

- `total_mv`：总市值
- `debt_to_eqt`：产权比率/杠杆约束
- `roa`：总资产回报率
- `roic`：投入资本回报率
- `pe`：市盈率
- `pb`：市净率
- `ps`：市销率
- `dv_ratio`：股息率
- `close`：股价上限过滤

## 3. 频率设计

- **数据更新频率**：日频
- **调仓观察频率**：季度观察
- **组合调仓频率**：当前代码逻辑以季度节点进行选股与调仓判断
- **自动化运行频率**：GitHub Actions 每个工作日运行一次
  - 配置文件：`.github/workflows/daily-update.yml:1`
  - 当前计划时间：工作日 UTC `11:00`，对应北京时间 `19:00`

换句话说，这个项目是一个：

- **日频更新数据与报告**
- **低换手的基本面选股策略**
- **以季度调仓为核心节奏的股票组合系统**

### 具体调仓日期

按当前代码逻辑，调仓锚点采用自然季度末，具体为每年的：

- `3月31日`
- `6月30日`
- `9月30日`
- `12月31日`

如果上述日期不是交易日，则通常以**下一个交易日**执行实际调仓。

以 `2026` 年为例，对应调仓日期可写为：

- `2026-03-31`
- `2026-06-30`
- `2026-09-30`
- `2026-12-31`

## 4. 组合与基准

- **组合类型**：股票多头组合
- **默认单次资金使用参数**：`per_trade=0.05`
- **目标持仓数**：约 20 只左右（参数中可调整）
- **对比基准**：
  - 沪深 300
  - 中证 500
